## Purpose

This gap analysis assesses SecureTech Inc.'s current compliance posture against NIST CSF 2.0, SOC 2, and ISO 27001:2022. It identifies what controls are implemented, what's partially in place, and what's missing — then recommends remediation priorities.

---

## Assessment Scale

Use this scale for each control area:

| Status | Meaning |
|---|---|
| **Implemented** | Control is fully in place, documented, and operating effectively |
| **Partially Implemented** | Control exists but has gaps (missing documentation, inconsistent enforcement, limited scope) |
| **Not Implemented** | Control does not exist or is not operational |
| **Not Applicable** | Control is not relevant to SecureTech's environment (justify why) |

Common gap patterns for startups:
- Policies exist informally but aren't documented → Partially Implemented
- Access reviews aren't done regularly → Partially Implemented
- No formal vendor risk management → Not Implemented
- No business continuity plan → Not Implemented
- Incident response is ad-hoc → Partially Implemented
- Logging exists but isn't centralized or reviewed → Partially Implemented
- Employee training is informal → Partially Implemented
